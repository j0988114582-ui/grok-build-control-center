### 額度 UI 文案（必須實作・可測）

當 `isUnifiedBillingUser === true` **或** `productUsage` 為空／缺分項時：

**主文案（繁中，定稿方向）：**

> 此帳號為**統一週額度**，總用量已涵蓋 **Build、Imagine（Image）、API**。服務未提供分項百分比，故顯示「—」，並非讀取失敗。

- 總額度環：**繼續顯示** `creditUsagePercent`  
- 三環 Build／Imagine／API：維持 `—`，**旁或 popover 顯示上述說明**  
- **禁止**：把總額度複製成三環 %；禁止改顯示 0%  
- 若未來 live 出現真實 `productUsage`：仍優先顯示分項 %；統一文案僅在無分項時出現  

**測試 T-Billing-1**：mock `isUnifiedBillingUser: true` + 空 `productUsage` → 總額有數字 + 可見統一週額度說明 + 三環為 —。  
**測試 T-Billing-2**：mock 有 `GrokBuild` 等 productUsage → 顯示分項 %，不強制顯示「未提供分項」主文案。

## 5. 技術要點（實作對齊）

### 5.1 插話

- Extension method：`x.ai/interject`  
- Params：`sessionId`, `text`, 可選 `interjectionId`, `content`  
- 成功：`status: "queued"`；**禁止**為插話呼叫 `session/cancel`  
- 立刻改做：先 `session/cancel` 再 `session/prompt`  
- live 對目前 grok 版本 smoke 一次；method not found 時 UI 降級提示  

證據：`xai-org/grok-build` `extensions/interject.rs`；研究檔 `research/grok-build-gui-integration-plan.md`。

### 5.2 額度

- 讀取：既有 `_x.ai/billing` → `normalizeBilling`  
- UI：`QuotaRings` + 統一文案（§2）  
- 保留 `productUsage` 真實分項路徑（測試 fixture 已有 GrokBuild 等）  

### 5.3 程序生命週期

- 參考 plugin-cc Windows 終止樹  
- 退出／disconnect／換 executable 皆應清子程序  

### 5.4 媒體

- 目錄：`%TEMP%\grok-build-gui-paste\`（既有清理 7 天）  
- 拖放與 paste 共用 save 管線
