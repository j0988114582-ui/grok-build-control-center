### 2.1 Renderer ownership invariant（硬）

**所有 `src/renderer/**` 變更**——含 JSX/DOM、CSS、frontend state、disabled／notice／action、renderer helper——**一律由 AGY 實作**。

- Executor **只**寫：`src/main/**`、`src/preload/**`、`src/shared/**`（契約與純邏輯）、`tests/**`、`scripts/**`、docs／version／CHANGELOG。  
- **Renderer 有 bug → 重新 dispatch AGY 修**，執行線不得「順手改 UI」。  
- F1 按鈕／notice、F2 搜尋框行為呈現、T1 範本 chips、T5 disabled 態：**前端由 AGY 接契約**。  
- 執行線可先寫 **shared readiness / search helper** 與 **測試骨架**；AGY 接線到 UI。

### 2.2 One-writer handoff（禁止同 worktree 搶寫 renderer）

單波 = **有序 handoff**，不是兩人同時改同一 worktree 的 `App.tsx`。

```
[1] 執行線：F3 有界 capability probe、F1 reveal 後端契約、T readiness 契約 + 測骨架
[2] 執行線 commit clean handoff → 停止一切 renderer 寫入
[3] AGY 獨佔 renderer：V1–V9 + T1/T5 前端 + F1/F2/F3/F4 前端
[4] 執行線：整合、全測、T4 live smoke（不改視覺，只修 main/shared/test）
[5] AGY：renderer nits（截圖後偏差）
[6] 執行線：verify + smoke:ui +（若出 installer）package → Codex 終審
[7] 無 P0 → 標「已交付」→ 已授權則 push 0.6.1
```

- 若必須平行：**隔離 worktree／branch**，禁止兩邊直接寫同一檔。  
- AGY：`model-exec.exe agy --allow-unbounded --write`，workdir = `_upstream`（handoff 後）。  
- 草稿僅視覺參考；**禁止**把 mock 百分比當 billing 真值。

#### 3.2.1 T4 Live smoke（必須重疊）

- 同一 Grok ACP process／connection。  
- 成功 load A、B 後，**兩個最小 prompt 時間重疊**。  
- 驗：event 依 sessionId；runningMap 不互污；**cancel A 不終止 B**；各自 stopReason。  
- 記錄 CLI 版本、時間戳、結果。  
- **站主 GO = 明示同意此額度消耗（約 2–3 次最小 prompt）。**

#### 3.2.2 T5 Readiness

- per-session readiness **綁定 ACP connection generation**。  
- 本 generation 內，session 在下列任一成功後視為 ready：  
  - **`session/new`（createSession 成功）**，或  
  - **`session/load`（loadSession 成功）**。  
  （二審 Codex：不可只認 load，否則新建對話會永久被 handler 拒絕。）  
- 僅 ready 的 session 可 send／interject／do-now／cancel。  
- load／create 進行中、失敗、disconnect 後：**UI disabled + handler 拒絕 + 明確 notice**。  
- disconnect **立即**使全部 readiness 失效（含剛 create 的 session）。  
- **測試必含**：新建 session 可送出；stale generation 的 session 仍拒絕。

#### 3.2.3 T6 Reconnect

- 前 snapshot：`slots`、`activeId`、`focusId`。  
- 全部 readiness 失效後 reload。  
- **不得**用會把 focus 最後停在 peer 的互動 load 路徑而不恢復；完成後 **原子恢復 snapshot**。  
- 部分失敗：隔離失敗 pane，保留成功 peers，報告結果。  
- 全部 reload 完成前禁止 prompt-capable actions。

#### T2 必含案例

- load-in-flight 不可送出  
- A/B load 反序完成仍保留最後使用者焦點  
- disconnect 使 readiness 失效  
- reconnect 保留 slots／active／focus  
- 單一 peer load 失敗不污染其他 pane  
- Esc 僅取消 **focused** running pane  
- 兩格獨立 draft；anyRunning 鎖權限模式

## 7. 技術要點

### 7.1 F1 匯出揭示（安全）

- **禁止**通用「任意 path reveal」IPC。  
- main 僅在 **本次成功 export** 後簽發 **token 或 allowlisted 絕對路徑**。
