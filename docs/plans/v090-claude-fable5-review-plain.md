# v0.9 手機遙控體檢報告（白話版）

日期：2026-07-19。這一版的重點功能是「用手機遙控電腦上的 Grok」，我把電腦端、手機端、還有兩邊溝通的管線整個檢查了一遍，找到 11 個問題並且全部修好、測試通過。

## 修了什麼（用生活比喻）

**1. 說好能寫 12,000 字，實際上一萬字出頭就被退件**
手機輸入框說上限 12,000 字，但伺服器的「信箱投遞口」只有 32KB 大。中文字比較「胖」（一個字佔 3 bytes），大約 10,700 字就塞不進去，會被莫名其妙拒收。→ 把投遞口加大四倍，現在 12,000 字中文穩穩送達。

**2. 長任務跑到一半，手機誤報「失敗」**
以前手機送出指令後，電話會一直「不掛斷」等到 Grok 整個做完才回覆。但中間的 Cloudflare 隧道規定通話最多約 100 秒，超過就強制掛斷——手機以為失敗了，其實電腦還在認真做。→ 改成「收到訂單立刻回條」：送出馬上回覆「已受理」，進度改由每 2.5 秒的狀態回報顯示，失敗會用通知告訴你。

**3. 權限詢問卡「兩邊不同步」**
Grok 問「可以執行這個工具嗎？」時，如果你在電腦按了同意，手機上那張卡片還會掛著最多 5 分鐘，再點會冒出英文錯誤；反過來手機按了，電腦的彈窗也不會關。→ 現在任何一邊回答，另一邊立刻收掉；回合結束也會自動清掉過期卡片。

**4. 手機開 YOLO 會「拔掉正在工作的插頭」**
切換 YOLO（一律核准）模式需要重啟與 Grok 的連線。電腦端有防呆：工作中不給切；手機端沒有——工作跑一半按下去，任務直接被砍。→ 手機端補上同樣的防呆，會請你先停止或等做完。

**5. 手機按「切斷」，電腦還以為連著**
手機登出後，電腦面板繼續顯示「已配對」。→ 現在會立刻變成「已過期」並跳通知。

**6. 電腦關掉遙控後，手機畫面「裝死」**
以前電腦端關遙控或斷網，手機只是靜靜停在舊畫面。→ 連續 3 次連不上就會在頂部顯示「連線中斷…畫面非最新」，恢復後自動復原。

**7. 手機畫面每 2.5 秒閃一下**
每次狀態更新都把整個畫面拆掉重蓋，造成閃爍、長按複製文字會被打斷。→ 內容沒變就不重蓋，畫面穩定了。

**8. 執行中還能按「送出」，按了必報錯**
→ 執行中送出鈕會變灰並顯示「執行中…」，要插話請用專門的「插話／排隊／立刻改做」按鈕列。

**9. YOLO 的 PIN 碼輸入很彆扭**
第一下點「開啟 YOLO」會直接跳紅字錯誤（其實只是要展開輸入框），而且 PIN 是明碼顯示。→ 改成兩步驟引導（先展開、再確認），輸入框改成密碼點點，成功後自動收合。

**10. 新建對話的路徑選單全變小寫**
`C:\Users\...` 顯示成 `c:\users\...`，看起來像壞掉。→ 保留原本大小寫（比對邏輯不變，安全不受影響）。

**11. 狀態更新偶爾被自家限流「絆倒」**
手機每分鐘約要 24 次狀態更新，但限流上限設 30，多按幾個動作就會被誤擋、畫面卡頓。→ 上限放寬到 60（防濫用仍在）。

另外：開啟 Quick Tunnel 時最長可能要等 2 分鐘驗證，以前只有一顆「啟動中…」按鈕，現在會顯示「隧道啟動驗證中（第 N/10 次）…」讓你知道沒當機。

## 第二輪（同日補完）：真通道實測＋掃尾

你問的「有沒有真的開通道、模擬手機連進去測」——第一輪沒有，第二輪補上了，而且變成專案的常設測試（`npm run smoke:remote-e2e`）：

**電腦真的開了 Cloudflare 通道，模擬手機從外網網址連進來，把整套流程走完，23 項檢查全數通過（跑了兩次、兩個不同的公網網址都過）**：掃碼配對頁 → 按 PIN 鍵盤配對 → 自動聚焦對話 → 送提示（**0.1 秒就收到「已受理」，即使任務跑 8 秒**——證明第一輪修的「長任務誤報失敗」在真通道下成立）→ 執行中送出鈕鎖定 → 回覆進畫面 → **12,000 字中文長提示過通道無誤** → 權限卡在手機上按「允許一次」往返成功 → 模型/模式選單套用 → 執行中開 YOLO 被正確擋下、閒置後開啟成功 → 雙重確認登出 → 電腦端同步顯示已過期。

上輪列的「待辦」也一次做完：

1. **手機可以「選」模型和工作模式了**——不用再打字輸入代號，選單直接列出電腦端目前可用的模型（含 reasoning effort 連動）；沒有資料時自動退回手動輸入。實測時還順帶抓到並修掉一個「套用模型後會把剛選好的工作模式洗掉」的重繪 bug。
2. **「立刻改做」的毫秒級競態**——改成先等舊回合的取消訊號落地再發新指令。
3. **幽靈對話清理**——建立失敗未落盤的對話 10 分鐘後自動消失；在電腦上刪除對話時，手機端的焦點、畫面、排隊也會一併清乾淨。
4. **兩邊防呆對齊**——安裝 CLI 或切換帳號進行中時，手機端操作會跟桌面一樣被擋下，不會互相打架。
5. **畫面收尾**——對話流裡的「running/completed」英文卡片改成「─ 回合開始／完成 ─」的細字標記；底部輸入區改為不透明，滑動時不再跟選單疊影。
