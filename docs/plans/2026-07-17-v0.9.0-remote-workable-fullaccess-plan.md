## 1. 與 0.8.0 R-SEC 的刻意差異（契約修訂）

| 0.8.0 | 0.9.0 |
| --- | --- |
| Remote **禁止** YOLO | Remote **允許** YOLO；**僅開 YOLO 要 PIN**；可只關 YOLO 留 Remote |
| Idle 30 分 + 絕對 4 小時 | **無 idle 斷線** + **絕對 72 小時**；手動重登重算 |
| Session 列表預設無 cwd | 列表**可含 cwd**（自用） |
| 遠端不可建 session／切模型 | **開放**（cwd 僅聯集；模型／模式走既有 ACP） |
| 遠端不可改權限模式 | **可**自手機 setPermissionMode（開 YOLO 走 PIN 閘） |
| Quick = 實驗 | **維持實驗**；文案誠實；書籤可壞 |

**未放寬（仍 Must）：**

- HTTP **僅 127.0.0.1**；不 `0.0.0.0`
- Host/Origin allowlist；無 CORS；mutation 要 JSON + `X-Grok-Remote`
- Cookie HttpOnly（HTTPS 隧道時 Secure）SameSite=Strict Path=/api；token 不進 JS storage
- Pairing fragment → replaceState → POST；secret ≥128-bit
- 伺服端只存 token hash；單活躍裝置預設
- permission.respond **物件級**綁定（若遠端仍允許手機回覆權限；YOLO 開啟時桌面 CLI 可能不再彈權限）
- 關 Remote：殺 tunnel + 關 HTTP + 作廢全部
- App 重開全失記憶體狀態 → 必須重配對
- 誠實隱私：Cloudflare 可處理 HTTP 內容
- 不做遠端任意 shell API（仍只經 Grok prompt／工具）

## 2. 安全契約補充（0.9.0 新增）

### 2.1 時鐘

| 規則 | |
| --- | --- |
| `REMOTE_SESSION_ABSOLUTE_MS` | **72h** from successful pair / successful re-pair |
| Idle | **不撤銷**（不設 idle timeout，或 idle = absolute） |
| 手動 logout / disable Remote / 桌面切斷 | 立即撤銷；下次 pair **新 72h** |
| App 重啟 | 記憶體清空 = 必須重配對 |
| 到期 | 踢下線；API 401；桌面 banner `expired`；不自動換碼 |

### 2.2 YOLO ↔ Remote（修訂版 · Fable E1/E8）

| 規則 | |
| --- | --- |
| 啟用 Remote | **計畫定案：若桌面已是 YOLO → 桌面確認一次風險後可 start**；否則 ask 下可 start。配對後亦可於手機 elevate YOLO（須 PIN）。 |
| 手機開 YOLO | 須 **有效 remote session cookie** + **PIN 驗證** + rate limit（E7）；PIN hash 在 pairing 成功後**仍保留至 Remote disable／72h／全撤銷**（非僅 pairing 窗）；plaintext PIN 僅桌面顯示 |
| 手機關 YOLO | 無 PIN；`setPermissionMode('ask')` + 既有重連語意；**Remote session 保留** |
| 桌面開／關 YOLO 而 Remote 已開 | **允許**；**不**強制撤銷 Remote |
| **模式切換不撤銷（E1 · P0）** | permission-mode **任何方向**（ask↔always-approve）、**任何發起者**（手機／桌面）的切換，**不得** `revokeAll` remote session/cookie、**不得**清 pairing pinHash、**不得**重置 72h 時鐘。0.8.0 `onPermissionModeChanged` → revokeAll **語意廢除** |
| 0.8.0 互斥殘留（E3） | `remote-yolo-mutex.ts` 的 `canEnableYolo(remoteActive)` / `canEnableRemote(yolo)` **改寫或刪除**；`index.ts` IPC、`remote-controller` enable/handlePair/handlePrompt 的 `yolo_conflict` 硬拒、`App.tsx` 閘 **全部對齊新約**；測試同步改寫 |
| CLI 套用 | always-approve 需重連 agent；重連期間 snapshot 標 busy；**重連後 main 自動恢復 focus session ready**（E2） |

### 2.3 PIN 生命週期（elevation · E7）

```
Remote enable → openPairing → 桌面顯示 PIN + QR
成功 pair → 關閉「新設備配對」但 **保留 pinHash 至 remote 生命週期結束**
PIN 用途：
  (1) 首次 pair
  (2) elevate-yolo（手機開 YOLO）
  (3) regenerate 後新 PIN 取代舊 hash
elevate 失敗計數 **獨立於** pairing.failures
每 token rate limit（建議 5 次 / 10 分）
5 次 elevate 失敗 → **鎖 elevation** 至桌面 regenerate PIN；
  session 仍在（ask 模式可繼續用）；snapshot 含 elevationLocked 供手機顯示
```

### 2.4 高風險能力 ACL

| API | 條件 |
| --- | --- |
| `/api/prompt` | cookie + focus session **ready** + rate limit；**YOLO 下仍允許**（非 yolo_conflict） |
| `/api/interject` | 同上 + running |
| `/api/do-now` | 同上 + running |
| `/api/queue` | cookie + focus；**main 側單槽**（E9）；與桌面 renderer 佇列兩寫 → **最後寫入者勝** + provenance 顯示 |
| `/api/cancel` | cookie + focus |
| `/api/session/focus` | cookie；sessionId **∈ 當下 listSessions**；見 §2.4.1 |
| `/api/session/create` | cookie；cwd ∈ **請求當下** cwd-union（E4）；見下 |
| `/api/session/list` | cookie；DTO 含 id/title/cwd/running；**可併入 snapshot** 省 rate |
| `/api/model` `/api/mode` | cookie + focus ready |
| `/api/yolo/enable` | cookie + PIN + 非已 YOLO + elevate rate limit + 未 elevationLocked（E7） |
| `/api/yolo/disable` | cookie；無 PIN；**不撤銷 remote** |
| `/api/permission/respond` | cookie + 物件綁定；YOLO 開啟時 pending 應為空 |
| 上傳 multipart | **不實作** |

#### 2.4.1 focus→ready 生命週期（E2 · P0）

| 規則 | |
| --- | --- |
| 權威 | **main** 執行 load / markReady；renderer **僅 UI 對齊**，不承擔 ready 責任 |
| `/api/session/focus` | 設 focusId；若未 ready：main 以 session summary 的 cwd 直接 `loadSession` + `markReady`；期間 snapshot `focusStatus: 'loading' \| 'ready' \| 'error'` |
| prompt 於 loading | 回 `not_ready` + 中文「對話載入中」 |
| ACP 重連後（含 YOLO 切換） | **main 自動**重載 **當前 focus** session 並 markReady；不得只重載「桌面舊 active」而忽略手機 focus |
| 雙寫 | 桌面 active 變更推 focus、手機 focus 推桌面：以**最後寫入者**為準；可帶 provenance notice |
| 整合測試（[2] 驗收） | elevate → reconnect → focus 自動 ready → 同一 cookie prompt 成功 |

#### 2.4.2 cwd-union create（E4）

- server **每次請求**重算 union：`listLocalSessions` → cwd 集合（磁碟 summary，不信任 client）
- `path.normalize` 後 **Windows 大小寫不敏感精確比對**；拒絕子路徑、相對路徑、空 union
- 成功後 main：`markReady` + `previewRoots.setSessionCwd`（與桌面 create 副作用對齊）

### 2.5 Quick Tunnel 與書籤

- 每次 Remote enable + Quick：新 URL 機率高；桌面大字提示「書籤請更新」
- **Quick URL 變更 = 新 host → host-only cookie 自然失效 → 需重配對**（寫明，無遷移）
- 桌面一鍵「複製公網配對 URL」
- QR：**本機** `qrcode` npm（或同等）生成；**零網路**；禁止第三方 QR API
- Health：公網 Host **先**入 allowlist 再 nonce health；失敗回滾 allowlist
- cloudflared：不接受 renderer 任意 path；checksum 完整釘選非 0.9 Must（誠實缺口）

### 2.6 隱私與自用聲明

- UI／CHANGELOG／Remote 面板：**單人高風險遠端控制**；72h；Quick 無 SLA；隧道供應商可見內容；YOLO = 工具自動核准。
- **威脅模型（Fable）**：手機遺失且已 YOLO → 竊者無 PIN 亦可能有至多 72h 全權；緩解 = 桌面切斷／regenerate、72h 絕對、手機 logout（二次確認）。
- 不做多租戶、不做帳號系統。

## 4. 手機 UX（書籤 + 效率）

### 4.1 資訊架構

1. **配對頁**：PIN 數字鍵盤、錯誤提示、無 secret 顯示  
2. **主殼**（配對後）：  
   - 頂欄：連線剩餘時間（72h 倒數）、YOLO 徽章、切斷  
   - **Session 抽屜／列表**：標題、cwd 短顯、running 點；點選 = focus  
   - **Transcript（E6）**：0.9 **僅記憶體 tail**（上限 **120 項 / 64KB**）；工具卡僅 title+status；**不做**磁碟歷史無限載入  
   - **Composer**：大觸控區；送出／停止；prompt 上限可 8–16k（body cap 內）  
   - **執行中工具列**：插話、排隊下一輪、立刻改做  
   - **溢位選單**：模型、工作模式、新建 session（cwd 列表）、YOLO 開／關  
   - **切斷（E5）**：**二次確認**；文案「切斷後需回電腦重新配對」  

### 4.2 效率原則

- 主要操作 ≤2 點達 composer  
- 44px 觸控目標  
- 繁中  
- `meta theme-color`；viewport-fit  
- 不依賴剪貼簿完成配對（PIN 手輸；URL 可書籤）  
- 書籤壞掉：首屏說明「請回電腦更新連結」  

### 4.3 不做

- Service Worker 離線（可選 P2）  
- 推播  
- 原生安裝
