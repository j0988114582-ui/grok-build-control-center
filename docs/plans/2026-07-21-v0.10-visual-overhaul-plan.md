# v0.10「Obsidian Voyage 黑曜遠航」視覺總改版計畫

日期：2026-07-21。目標：修死冷啟白灰凍結 bug；深色優先重構整體視覺為「精緻・商業・奢華・星際遨遊」；icon 與全部視覺元素升級；亮色其次。

## 0. 已確診的冷啟 bug（Phase 0 先行修復）

**症狀**：每次開啟程式，主畫面是白灰放射光而非深色星空；進工作臺切亮色→切回深色才恢復。

**根因**（`src/renderer/src/fx/starfield.ts`）：
1. 開場「launch」動態設定 `flash: 0.9`，fog shader 疊 `u_flash * vec3(.32,.48,.62)` 的**淡藍灰全屏光**，設計上 1650ms 後衰減。
2. 渲染迴圈唯一的續命是 `requestAnimationFrame` 鏈；`webglcontextlost` 會取消迴圈，restore 事件在冷啟 GPU 情境可能不來；`document.hidden` 停迴圈也依賴 visibilitychange 重踢。**迴圈一停，最後一幀（正處於閃光高峰）永久凍結**。
3. 主程序 `disable-gpu-shader-disk-cache` + `disk-cache-size=1`（為多開防噪）→ 冷啟 shader 全重編，放大凍結機率。
4. 主題切換會走 `visible=false→true` 銷毀重建引擎（GPU 已暖）→「切亮再切暗就好」的原因。

**修復架構**（不只補這一例，消滅整類問題）：
- 開場改「**曲速抵達**」：由全暗漸現星流（亮度只往上限 0.18 的金藍色調走），**永不出現大面積亮灰疊色**；凍結最壞情況＝深色畫面。
- **自癒 watchdog**：每 2s 檢查「應該在動卻超過 1.5s 沒新幀」→ 重踢 rAF；context-lost 超過 3s 未 restore → 整個引擎重建（含 shader/buffer）；visibility/focus/resize 都重踢。
- **CSS 永久兜底**：`.starfield-canvas` 底下鋪一層純 CSS 深色星空（radial gradients 靜態星點），引擎徹底死掉畫面仍正確。
- **冷啟 E2E 驗收**：新增 smoke——啟動打包版 → 3 秒內連續截圖 → 斷言畫面暗色像素占比 > 85%、無亮灰凍結幀。

## 1. 設計語言「Obsidian Voyage」

- **色彩**：黑曜深空底（#030711 系階梯）、**香檳金**主張色（現有 #e9ad47 延伸出金屬漸層 #f5d9a0→#e9ad47→#a06b1f）、冰藍輔助（資料/連線）、紅移只留給錯誤語意。禁止大面積亮灰。
- **材質**：玻璃擬態面板（backdrop-blur 8–12px、rgba 深底、1px hairline 金/藍邊）、按鈕金屬鍍邊＋內陰影、卡片微噪點紋理。
- **動態紀律**：開場曲速序曲（≤2.2s、可跳過）、執行中星流加速、連線金色漣漪、錯誤紅移脈衝（沿用現有語意）、`reduced-motion` 全靜態單幀。
- **字體**：保留 IBM Plex Sans Variable＋Newsreader；強化標題字距與層級（hero 用 Newsreader italic 點綴奢華感）。

## 2. 技術路線（三選項，建議 A＋B）

| 路線 | 內容 | 優點 | 代價 |
|---|---|---|---|
| **A. 升級自製 WebGL 引擎**（基底，強烈建議） | 多層視差星層（3→5 層）、程序化星雲色帶（分層 noise、金/藍雙色域）、偶發流星、景深光斑、主題調色盤參數化（dark/light 各一組，修掉引擎不吃 theme 的缺口） | 最輕量、確定性、既有測試資產可沿用、不加依賴 | 視覺上限低於真 3D |
| **B. three.js/R3F 3D 點綴**（hero 專用） | 依賴已在專案內（目前 0 使用）。只用在歡迎頁 hero：3D 金環星門／儀表環、滑鼠視差、instanced 粒子 | 奢華感上限高；範圍鎖 hero 不拖累全域效能 | bundle +~600KB；需效能預算管理 |
| **C. 影片背景**（可選外掛） | 使用者提供無縫 loop 影片，用於歡迎頁 ambience（非全域） | 電影感最快 | 無法隨狀態互動（加速/紅移做不到）、縮放裁切、耗電；規格要求：2560×1440+、10–20s 無縫 loop、webm/H.264、<15MB、深色基調 |

不建議全域改 R3F：背景層要求「常駐、低功耗、絕不能死」，自製引擎＋watchdog 比 React 掛載樹更可控。

## 4. 分階段執行

- **Phase 0（hotfix，最先）**：修凍結＋開場去白閃＋冷啟 E2E。可獨立出 0.9.1 patch。
- **Phase 1**：深色主題引擎升級（路線 A）＋設計 token 重整（colors/borders/glow 統一變數）。
- **Phase 2**：icon＋品牌資產全套。
- **Phase 3**：元件精緻化——buttons/pills/modals/session 列表/QuotaRings/命令面板/歡迎頁 hero（路線 B 或 C 在此落地）；手機遙控頁同步套用 token。
- **Phase 4**：亮色主題「晨光星雲」（淡金＋米白＋淺藍；星空改極淡粒子）。
- **Phase 5**：驗收閘——axe serious/critical=0、reduced-motion 全靜態、效能預算（idle GPU <8%、無記憶體上升斜率）、全測試綠、冷啟 E2E、打包實跑。

## 5. 定案（2026-07-21 使用者拍板）

1. 背景路線：**A+B**——自製引擎大升級為全域背景；three.js/R3F 只做歡迎頁 3D hero（lazy chunk、WebGL/reduced-motion 自動降級）。
2. icon 方向：**曜石艦首稜鏡**（黑曜多面體＋金稜線＋冰藍引擎光）；先出 3–5 張概念圖供挑選細節。
3. 影片素材：**不用**，全部程式生成。

執行順序：Phase 0+1 同批落地（同檔案）→ 冷啟 E2E＋smoke 截圖驗收 → 3D hero → icon 概念圖 → 使用者回饋後進 Phase 2 整合與 Phase 3 全元件精緻化 → Phase 4 亮色。
