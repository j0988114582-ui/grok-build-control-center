## 1. 一句話目標

讓日常使用不再卡在「找對話、關側欄、輸入框搶畫面、權限只能逐次點」——在**不破壞 ACP 邊界與安全誠實性**的前提下，交付一版可感知的 UX 升級。

建議版本號：**0.4.0**（使用者可見功能增量；非僅文件）。

## 2. 審查糾錯（避免帶錯假設開工）

| 說法 | 真相 |
| --- | --- |
| 「權限只留單筆、會被覆蓋」 | **錯**。已是佇列：`permissions[]`、顯示「還有 N 項待決」、`permissions[0]` 出列。 |
| 「要靠 session/set_config_option 調權限」 | **不可行**。本機 ACP 回 `Method not found`；`session/new` 亦常不帶 `modes`/`configOptions`。 |
| 「Safe-Auto 用 `--approve-read-only`」 | **未證實**。Gemini 臆測；正式旗標以 CLI 為準：`grok agent --always-approve`、TUI `/always-approve`、`/auto`、config `[ui] permission_mode`。 |
| 批次刪除是 bug | **功能缺口**，非回歸 bug。CLI 仍為單筆 `grok sessions delete`。 |
| ACP 可貼圖 | **否**。實測 `promptCapabilities.image=false`、`audio=false`、`embeddedContext=true`。 |

## 4. 權限模型：可行設計（U6）

### 現實約束

1. **逐次核准**（現有）：ACP `requestPermission` → modal → `respondPermission`。  
2. **模式切換**：CLI 層 `--always-approve`；TUI `/always-approve`、`/auto`；config `permission_mode` / `yolo`。  
3. **ACP 動態 set_config**：目前 **不可用**。  
4. 官方文件：`bypassPermissions` / always-approve 仍尊重 deny rules / hooks；但風險仍高（任意 shell）。

### v0.4 建議 MVP（兩層，不裝假）

**A. 全域／連線級「權限模式」（設定頁 + 狀態列）**

| UI 選項 | 行為 | 實作 |
| --- | --- | --- |
| 每次詢問（預設） | 維持現狀 | `buildAgentArgs()` 不加 flag |
| 一律核准（YOLO） | 跳過多數 permission 提示 | 重連 ACP 時附加 `--always-approve`；**紅色二次確認**；標題列常駐警告 |

- 變更模式 = **重連 agent**（停止舊 process → 新 args 啟動）。  
- 不宣稱支援未驗證的 flag。  
- 可選：若 `availableCommands` 含 `always-approve`，額外提供「對目前 session 送 `/always-approve` 文字」作為不重連捷徑（需 live 驗證後再上；失敗則隱藏）。

**B. 逐次 modal 小強化（低成本）**

- 保留佇列。  
- 選項文案更白話（允許／拒絕／允許類似…依 agent 給的 option）。  
- 可選「拒絕剩餘佇列」按鈕。

**明確不做（0.4）**

- 自造 per-tool allowlist 引擎（應走 Grok hooks / config，GUI 不平行發明）。  
- 未驗證的 Safe-Auto 旗標。

## 5. Pin 設計（U5）

| 項目 | 決策建議（預設，可改） |
| --- | --- |
| 儲存 | `AppSettings.pinnedSessions: string[]`（electron-store），**不**改 Grok `summary.json` |
| 排序 | 側欄頂部獨立分組「已釘選」；組內仍可顯示專案名小標；其餘維持專案分組 |

| 範圍 | **全域 pin**（跨專案列表頂部）— 符合「一直找特定對話」 |

## 6. v0.4.0 範圍切線

### Must（本版一定交）

1. **U1** 側欄永遠可重開（empty-state + 窄 rail 或 titlebar 浮鈕 + 可選快捷鍵）  
2. **U2** Composer 固定高度、內容內捲、限制 `resize`  
3. **U3** 專案標題字級／對比上調；對話名降層級  
4. **U5** Pin session（settings 持久化 + 頂部分組）  
5. **U6** 權限模式：設定 + 狀態警示 + `--always-approve` 重連（含二次確認）

### Should（時間夠就進 0.4；否則 0.4.1）

6. **U4** 批次刪除（多選 + 確認 + 逐筆 CLI + 結果摘要）  
7. 刪除／批次刪時清理 orphan `sessionTitles`/`drafts`/`pinnedSessions`  
8. 快捷鍵：toggleSidebar；文件 `?` 表更新  

### Later（0.5+ / 協定成熟後）

9. **U7** 貼圖：ACP 未開 image 前 → 自動落檔 + 路徑／`resource_link`；開 image 後再做 chip  
10. 影片／音訊多模態輸入  
11. Mode 下拉穩定化（需 CLI 穩定回 modes 或替代 API）  
12. 安裝腳本 hash、程式碼簽章（對齊 ROADMAP Release trust）  
13. App.tsx 大拆分、多選批次匯出等

## 7. 建議實作分期（工程順序）

| 階段 | 內容 | 驗收 |
| --- | --- | --- |
| **M1 修阻塞** | U1 側欄、U2 composer、U3 標題樣式 | 主頁收合可重開；長文不擠 transcript；專案名可讀 |
| **M2 找得到對話** | U5 Pin | 重開 app 後 pin 仍在；頂部分組正確；刪除清 pin |
| **M3 權限可調** | U6 模式設定 + 重連 + 警告條 | 預設仍逐次問；YOLO 需二次確認且徽章可見；重連後不炸 session 列表 |
| **M4 清理效率** | U4 批次刪（若進版） | 多選刪 N 筆；部分失敗有報告；active 被刪回 empty-state |
| **M5 包裝** | CHANGELOG、README 短述、測試、package | `npm test` / lint / typecheck；可選 smoke |

**預估（solo）**：M1+M2+M3 ≈ 一個密集工作段；M4 另半段。不綁死日曆。

## 8. 測試最低門檻

- 單元：`session-groups` pin 排序；`normalizeSettings` 含 `pinnedSessions`；`buildAgentArgs` 依權限模式  
- Renderer：側欄 empty-state 展開；composer 高度 class；pin 按鈕；權限模式確認 dialog  
- 回歸：既有 permission 佇列、單刪、IME、drafts  
- Live（可选手動）：YOLO 重連後工具不再彈權限；關閉 YOLO 後恢復詢問
